# DeskMascot-demo
DeskMascotの紹介・デモページ

スマホのホーム画面や他アプリの上に、自分で用意したキャラクターを常駐表示できる Android アプリです。キャラクターは画面内を歩き回り、話しかけると AI が返事をし、なでると喜怒哀楽の反応を返します。Kotlin + Jetpack Compose で個人開発しました。

> 個人開発・学習用に制作。実機（Xperia）で動作確認済み。

## 主な機能

<!-- 各機能の動きは、下の該当セクションのGIF/スクショで確認できます -->

- **オーバーレイ表示** — 他アプリの上にキャラクターを重ねて表示し、自由に歩き回らせる。ドラッグで好きな位置に移動できる
- **AI 会話** — Gemini API（無料）または OpenAI API でキャラクターとチャット。会話履歴を文脈として保持し、キャラ設定（口調・性格）に沿った返答をする
- **好感度システム** — なでたり会話したりで好感度が変化（-200〜1000、5段階）。好感度に応じて表情・口調・態度が変わり、低いと拗ねて背を向ける
- **お触りモード** — キャラを拡大表示してタップ／なでなでで反応。喜怒哀楽をランダムに返す
- **天気・カレンダー連携** — Open-Meteo API で現在地の天気を、端末カレンダーから当日の予定をキャラが教えてくれる
- **通知連携** — 他アプリの通知が来るとキャラが吹き出しで知らせる
- **衣装・時間帯対応** — 時間帯（朝/昼/午後/夜/深夜）ごとに衣装やセリフを切り替え。深夜は寝て、構うと不機嫌になる（不機嫌な時間帯は設定可能）
- **キャラの共有** — 作ったキャラ（画像・セリフ・衣装）を 1 ファイルにまとめて他ユーザーに配布。個人情報は書き出し時に除外
- **完全カスタマイズ** — 画像・名前・セリフ・大きさ・歩く速さなどをプリセットとして複数保存・切替

すべてユーザー提供の画像・テキストで動作し、キャラクターの見た目やセリフはコードに固定していません。

## 技術スタック

| 領域 | 使用技術 |
|---|---|
| 言語 | Kotlin |
| UI | Jetpack Compose (Material3) |
| 常駐処理 | Foreground Service + `TYPE_APPLICATION_OVERLAY` |
| 非同期 | Kotlin Coroutines |
| データ保存 | DataStore (Preferences) + kotlinx.serialization |
| 画像 | Coil（静止画／GIF・アニメWebP対応）、ImageDecoder |
| 通信 | OkHttp（Gemini / OpenAI / Open-Meteo の各 REST API） |
| 通知 | NotificationListenerService |
| 共有 | FileProvider + ZIP パッケージング |

最小 API 28 (Android 9) / ターゲット API 35。

## 設計上の工夫

**オーバーレイと吹き出しのウィンドウ分離**
キャラ本体と吹き出しを別々のオーバーレイウィンドウに分離し、吹き出しの表示・非表示でキャラ本体のレイアウトに影響が出ないようにしています。当初は単一ウィンドウ内に配置していましたが、吹き出しの出し入れでキャラが揺れる問題があり、ウィンドウ分離で根本解決しました。

**アニメーション画像の再描画対策**
GIF / アニメ WebP（`AnimatedImageDrawable`）はインスタンスを使い回すと再描画に失敗するため、静止画のみキャッシュし、アニメは表示のたびにデコードし直す設計にしています。クリーンインストール時のみ発生する描画不具合の原因を、キャッシュ状態の違いから特定しました。

**プライバシー配慮のキャラ共有**
キャラ共有時、API キー・居住地（天気の地点）・好感度・ユーザー名などの個人情報は「取り込み時に消す」のではなく「書き出し時にそもそも含めない」方式にしています。共有ファイルを直接開かれても個人情報が漏れない設計です。

**単一の時間帯ロジック**
朝/昼/午後/夜/深夜の判定を 1 箇所（`TimeKeys`）に集約し、セリフ・衣装・天気・カレンダー・深夜の挙動すべてがこれを参照するようにして、時間帯まわりの仕様の一貫性を担保しています。

**設定の後方互換**
`ignoreUnknownKeys` を有効にし、アップデートで設定項目が増えても旧バージョンの保存データを壊さず読み込めるようにしています。同一署名での上書き更新で、ユーザーの設定・好感度が保持されます。

## プロジェクト構成

```
app/src/main/java/com/example/deskmascot/
├── service/
│   ├── MascotOverlayService.kt    # 中核。オーバーレイ表示・歩行・モード遷移・吹き出し
│   └── MascotNotificationListener.kt
├── ui/
│   ├── MainActivity.kt            # ホーム画面
│   ├── SettingsScreen.kt          # キャラ設定
│   ├── ChatActivity.kt            # AI 会話画面
│   └── HistoryScreen.kt           # 会話履歴
├── data/
│   ├── PresetRepository.kt        # キャラ設定の永続化・好感度ロジック
│   ├── PresetShare.kt             # キャラの書き出し／取り込み
│   └── ChatHistoryRepository.kt
├── api/
│   └── GeminiClient.kt            # Gemini / OpenAI 自動切替
└── util/
    ├── WeatherHelper.kt           # Open-Meteo
    └── CalendarHelper.kt
```

## 開発について

個人開発で、仕様の検討から実機でのデバッグまで一通り行いました。特にオーバーレイ表示はウィンドウ管理の非同期性に起因する座標・タイミングの問題が多く、実機で症状を確認しながら原因を切り分けて解決しました。改善の過程は [CHANGELOG.md](./CHANGELOG.md) に記録しています（v1.0〜v7.8、70回以上の改善）。

AI（Claude）をペアプログラミング相手として活用し、設計の壁打ち・実装・デバッグを進めました。

## セットアップ

1. Android Studio で本リポジトリを開く
2. 実機（Android 9 以上）を接続してビルド・インストール ※オーバーレイの動作確認は実機推奨
3. アプリを起動し、設定画面でキャラクター（画像・名前）を作成
4. 「他のアプリの上に表示」の権限を許可
5. AI 会話を使う場合は [Google AI Studio](https://aistudio.google.com/) で無料の API キーを取得し、設定画面に入力

詳細な手順・権限まわりの注意は同梱の使い方ガイドを参照。
